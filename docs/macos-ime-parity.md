# macOS日本語IME互換性ループ

KarukanのmacOS版は、Apple標準の日本語入力と同じ予測可能性、視覚的な
安定性、キーボード操作を目標とする。モデル固有の機能は残すが、入力中に
ユーザーが見る状態や操作方法はプラットフォームの慣例を優先する。

## 反復手順

1. Appleの現行ガイドとTextEdit上の純正IMEで、対象操作を1つ観察する。
2. Karukanとの差を、再現手順とユーザー影響に分けて記録する。
3. 失敗するRustまたはSwiftテストを追加し、最小の実装で直す。
4. `./script/ime_quality_loop.sh --check` でRust、Swift、clippy、format、
   Thread Sanitizerを通す。
5. `./script/ime_quality_loop.sh --install` で署名済みアプリを入れ替え、
   TextEditと普段使うアプリで入力・変換・取消・候補選択を確認する。
6. 1つのまとまりとしてcommitし、復元点を残してから次の差分へ進む。

## 現在の基準

- キー入力直後は入力バッファどおりの読みを表示し、直前の変換結果を新しい
  読みの前へ継ぎ足さない。
- 候補は8行単位で移動し、数字、矢印、Space、Shift+Space、
  Control+N/P/R/V、Page Up/Downで操作できる。
- 文節は個別の下線で分け、選択文節は太い下線にする。
- 候補パネルはシステムのpopover素材、強調表示色、外観、透明度設定へ
  追従し、モデル名・処理時間・トークン数などの診断情報を表示しない。
- 変換候補の学習削除など、現在の選択に必要な操作だけを補助表示する。
- 候補パネルは現在の選択文節または入力カーソルへ追従し、文節移動や
  エディタのスクロール後も以前の画面位置に残らない。
- 候補パネル上のホイールまたはトラックパッド操作で前後候補へ移動でき、
  入力先アプリのキーボードフォーカスは奪わない。
- 複数ページの候補はテキストのページ番号ではなくmacOS標準のスクロールバーで
  位置を示し、スクロールバーのクリックやドラッグでもページを移動できる。
- ひらがな入力時のOption+8/9、Option+Shift+8/9、Option+;はApple標準と
  同じ日本語記号を入力し、それ以外のOption/Command操作はアプリへ渡す。
- 辞書とモデルの初期化は入力処理と分離し、起動直後もローマ字からかなへの
  規則変換を待たせない。初期化完了後、キー処理の合間に資源を反映する。
- 初期化中に確定した選択も一時保持して学習キャッシュへ合流し、モデルだけが
  読めない場合も、読み込み済みの辞書と学習データは破棄しない。
- 初期化完了前にIMEが終了しても、モデルを待たず学習ファイルだけを読み込んで
  起動直後の選択を安全に合流してから保存する。
- ホストアプリからの周辺文脈取得は最初のキー表示後に遅延実行し、アプリ固有の
  文書アクセス速度や定常ログが初打鍵の応答時間へ入らないようにする。
- 句点のない長文は、アイドル更新1回につきニューラル変換を1チャンクに制限し、
  150 msずつ入力処理へ譲りながら続きを更新する。複数チャンクの直列推論で
  キーイベントや1.5秒の応答期限を塞がない。各チャンクは一度だけ段階表示し、
  画面と確定対象を一致させたまま高速な全体書き換えを避ける。
- macOSの遅延ライブ変換は1チャンクを最大120かなまでまとめ、語の途中に
  固定長境界が入ることで長文の精度が落ちるのを抑える。軽量モデルでの実測は
  108かな約37 msで、通常の同期変換は30かな設定を維持する。
- かなだけの読みからモデルが推測した数字はライブ表示へ採用しない。直接入力した
  数字はそのまま保持し、読み上げ数字への変換はSpace後の候補で選択できる。
- 最後のキー入力からライブ変換開始までの待機は150 msとし、連続入力中の
  不要な推論を避けながら、250 ms待機時より初回表示を100 ms早める。
- 長文の後続チャンクは100 ms間隔で段階表示する。短いチャンクによる
  キー応答上限と一方向の安定表示を保ちつつ、句読点なし長文の完了を早める。
- エンジンが消費しないファンクションキー等では、待機中のライブ変換を
  取り消さない。入力を編集したキーとエンジン応答不能時だけ世代を更新する。
- 自動実機テストの入力ソース切替は、TIS上のID一致後も0.5秒待って再確認する。
  InputMethodKitの有効化前に先頭のキーが旧入力ソースへ漏れる競合を避ける。

## 実機検証

- `./script/ime_quality_loop.sh --install` はビルド成果物とインストール済み
  バイナリの一致、署名、起動、Karukanへの入力ソース切替を確認し、終了時に
  元の入力ソースへ戻す。
- 現在の入力ソースは
  `karukan-macos/scripts/select_input_source.swift --current`、Karukanへの切替は
  引数なしの同スクリプトで確認できる。
